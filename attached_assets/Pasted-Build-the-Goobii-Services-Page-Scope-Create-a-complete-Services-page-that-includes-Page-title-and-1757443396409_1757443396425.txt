Build the Goobii Services Page
Scope

Create a complete Services page that includes:

Page title and a short Sooftwash™ intro paragraph

A simple intro line and a follow-up instruction line

Four service boxes (use the pre-shared Goobii box shape; Replit knows this shape)

Each box shows: gold title, one-line description, Highlights on the left, Estimated duration on the right, and a short “Best for” line

Pressing a box opens a modal popup (covers most of the screen) with a smooth animation and shows a detailed process + placeholder illustrations/icons for products, tools, equipment

Bottom CTA with centered app icons/buttons

Tech

Next.js (App Router) + React + TypeScript

Tailwind CSS

Framer Motion for smooth transitions (respect prefers-reduced-motion)

Inline SVGs for icons/illustrations (placeholders)

Brand system (use exact tokens)
:root{
  --brand-primary:#006980;      /* main text & buttons */
  --brand-secondary:#003B5C;    /* contrast accents */
  --brand-bg:#FAFCFE;           /* background */
  --brand-pop:#FF8200;          /* pop/time */
  --brand-contrast-2:#DCC57F;   /* selection highlight (GOLD) */
}

4) Modal popup on press (smooth animation)

Build components/ServiceModal.tsx and components/ServiceCard.tsx

On press/click or Enter/Space, open a modal that covers most of the screen (max-width ~900–1100px, responsive)

Animation: fade + slight scale up (spring), backdrop fades in; respect prefers-reduced-motion (no scale, quick fade only)

Accessibility:

role="dialog", aria-modal="true", labeled by the service title

Focus trap inside modal; ESC closes; backdrop click closes; Return focus to triggering card on close

Modal content structure:

Title (service name) in gold and bold

Short purpose line (reuse the card description)

Process (numbered 1–4)

Used with three lists: Products · Tools · Equipment

Right side or bottom: placeholder illustration/icons (inline SVGs) for each list item
